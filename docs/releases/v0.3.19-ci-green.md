## What Ships

### CI Infrastructure Fixes

| Commit | Change |
|--------|--------|
| `523e9680` | Added `SUPABASE_SERVICE_ROLE_KEY` secret to GitHub repo and to the `Build` step env in both `e2e-smoke.yml` and `parity-gate.yml`. The `check-env.js` pre-build guard requires this key and was aborting both workflows silently. |
| `a15a4718` | Propagated `SUPABASE_SERVICE_ROLE_KEY` to the `Start server` step in `parity-gate.yml`. Without it, `next start` was crashing silently in the background, causing `wait-on` to time out. Also bumped `wait-on` timeout from 60s → 120s. |
| `d1548179` | Added `webkit` to the Playwright browser install step. Scoped the mandatory parity test run to `--project=desktop-chrome` (Ubuntu CI does not carry Safari/WebKit by default; all `mobile-safari` tests were failing in 3–4ms from missing binary). |

### Test Hardening

| Commit | Change |
|--------|--------|
| `d4f5b425` | Fixed non-deterministic `rejects invalid checksum` test in `displayId.test.ts`. The old test tampered the checksum digit with `'X'`, which is a valid alphabet character — if the real checksum happened to be `'X'` the tampered ID passed validation and the assertion failed. Fix: pick any alphabet character that provably differs from the real checksum. |
